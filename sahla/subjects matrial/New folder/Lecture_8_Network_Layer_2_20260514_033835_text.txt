Router Architecture Overview
A router performs two primary functions:
1. Routing (Control Plane)
This is handled by the routing processor.
What It Does:
e Runs routing algorithms or protocols such as:
e RIP (Routing Information Protocol)
¢ OSPF (Open Shortest Path First)
¢ BGP (Border Gateway Protocol)
¢ Builds and maintains the routing table.
e Determines the best path to each destination network.
e Sends the resulting forwarding information to the input ports as a forwarding table (FIB).
Position in Diagram:
e The routing processor is shown at the bottom.
e It communicates with the switching fabric to update forwarding information.
2. Forwarding (Data Plane)
This function moves packets from an incoming link to the correct outgoing link.
Where It Happens:
¢ Primarily inside input ports, switching fabric, and output ports.

3. Lookup, Forwarding, and Queueing (Network Layer / Decentralized
Switching)
This block performs the core routing decision.
Decentralized Switching
e Each input port can independently perform the forwarding lookup.
e The forwarding table is stored locally at the input port.
e This avoids sending every packet to the routing processor — improving speed.
What happens here:
a. Lookup
e The input port checks the destination IP address in the datagram.
e It uses the forwarding table (FIB) stored at the input port.
e Determines which output port the packet should go to.
b. Forwarding
e Prepares the packet for entry into the switching fabric.
e Sets metadata and routing info used by the fabric.
c. Queueing
e If packets arrive faster than the switching fabric can accept them:
e They are placed into an input queue.
e Needed because high-speed links can produce bursts of packets.
Why Decentralized Switching?
Goal: Process packets at line speed
e “Line speed” means the router must handle packets as fast as they arrive on the interface.
e If lookup were done centrally (by routing processor), performance would collapse.
e Decentralizing the lookup provides parallelism.

1. Memory Switching
e The CPU (routing processor) copies each packet from the input port into memory, then copies it back
out to the output port.
e Only one packet can be moved at a time because it uses a single shared memory.
e This method was used in early routers.
e Limitation: Slow — switching speed limited by memory bandwidth.
2. Bus Switching
e All input ports share a common bus.
e A packet is placed on the bus and every output port can listen; only the intended output port accepts it.
e Still only one packet can traverse the bus at a time (shared medium).
e Faster than memory-based, but still limited by the bus bandwidth.

What an Output Port Does
It typically performs three main functions:
1. Buffering / Queuing
2. Data-link processing (protocol, encapsulation)
3. Line termination / actual transmission onto the physical link
1. Buffering: Why It’s Needed
When packets exit the switch fabric (the internal network inside the router), they reach an output port.
Sometimes:
e Packets arrive faster than the outgoing link can transmit them
e OR bursts of packets arrive simultaneously
Since the outgoing link has a fixed transmission rate, any packets that can't be sent immediately must be
stored temporarily in a buffer.
Why buffering is required:
e Prevent packet loss when output link is temporarily overloaded
© Smooths traffic bursts
¢ Allows fair or priority forwarding if multiple flows share the same link
If the buffer fills up — packets get dropped (queue overflow), causing congestion.

2. Scheduling Discipline: Choosing Which Packet Goes Next
When multiple packets are waiting in the output queue, the router must decide which one to send first. This
is called packet scheduling.
Different scheduling algorithms exist, for example:
FIFO (First-In First-Out)
e Simplest: send packets in arrival order.
Priority Queuing
e Higher-priority traffic (e.g., VoIP) is sent before lower-priority traffic.
Purpose of scheduling:
e Enforce quality-of-service (QoS) guarantees

The diagram shows:

e Incoming datagrams from the switch fabric are stored if the link is busy.

The two images illustrate output port contention — a situation where multiple packets simultaneously
want to use the same output port in a switch/router.
Left Side: Time t (Contention)
e Several input ports each have a packet ready to move through the switch fabric.
e All of these packets are trying to go to the same output port.
e The switch fabric can deliver the packets to the output port’s queue, but the output link can transmit
only one packet per time unit.
e Therefore, only one packet is sent out immediately.
e The others must wait in the output queue, causing queuing delay.
Right Side: One Packet Time Later
e The output port has transmitted one packet.
e Now the next packet in the queue moves forward.
e The circled packet is the one that had to wait because it couldn't be transmitted in the previous time
slot.
e This waiting is the delay caused by contention.
e The output queue now holds the remaining packets waiting their turn.

Version Says which IP version is used. Here it is version 4 (IPv4).

Options Extra optional information (usually not used).

Step-by-Step Explanation
1. A big IP packet is created by the sender
e The sender PC creates one large IP packet (big red+blue block in the picture).
e This packet is too large for some network links.
2. Router sees the packet is TOO BIG for the next link
Every link has an MTU (e.g., 1500 bytes for Ethernet, 620 bytes for others).
If the packet is larger than the MTU — it must be fragmented.
Think of MTU as the maximum size of a truck allowed on a road.
3. Router breaks the packet into smaller fragments
e The image shows the big packet split into 3 small pieces (red/blue blocks).
e Each fragment gets its own IP header.
e Each has:
e same Identification number (ID)
e fragment offset (position)
e MF flag (More Fragments)
This allows the receiver to know:
which fragments belong together
@ what order they should be in
@ when the last fragment arrives

A subnet (subnetwork) is simply a smaller, logical section of a larger IP network.
It is defined by the subnet part of an IP address (the high-order bits), which is shared by all devices in that
subnet.
Key idea
Devices whose IP addresses share the same subnet portion can directly communicate with each other
without a router.
Only a switch or hub is needed inside the subnet.
How the picture explains it
In the diagram, you see three blue areas. Each area is a separate subnet:
e Subnet 223.1.1.x
e Subnet 223.1.2.x
e Subnet 223.1.3.x
Inside each blue area:
e All devices have IP addresses that start with the same subnet prefix, e.g., 223.1.1.*
e They can reach each other directly by sending Ethernet frames (layer-2 communication).
e No router is needed inside the subnet.
Between the blue regions:
e The router connects different subnets.
e Any communication between 223.1.1.x and 223.1.2.x, for example, must pass through the router because
their subnet bits differ.

Motivation
NAT allows an entire local (private) network to appear to the outside Internet as if it were using only one
public IP address.
This provides several key benefits:
1. Save Public IP Addresses
e Devices inside use private IP ranges (e.g., 192.168.x.x).
e The ISP assigns only one public IP address to the organization or home network.
e No need to request a block of public addresses.
2. Independence from ISP
e You can change ISP or public IP address without changing:
e Internal device addresses
e Network configuration
e Only the NAT router's public IP must change.
3. Internal Address Flexibility
e Network administrators can renumber internal IP addresses freely
e No impact on external communication.
4. Improved Security
e Internal devices are:
e Not directly visible
e Not directly reachable
e Only the NAT router is seen by the outside world.
e Acts as a basic security barrier against direct scanning or attacks.

1. IPv6 networks at the edges
Nodes B and E (and their routers) support IPv6.
2. Middle of the network is IPv4-only
Some routers support only IPv4 and cannot read or forward IPv6 packets.
3. A tunnel is created between IPv6-capable routers
Two IPv6 routers that are far apart create a logical “tunnel” through the IPv4-only region.

IPv4 routers do not understand IPv6, but they simply forward the outer IPv4 header.

The outer IPv4 header is removed (“decapsulation’), and the original IPv6 packet continues its journey.
