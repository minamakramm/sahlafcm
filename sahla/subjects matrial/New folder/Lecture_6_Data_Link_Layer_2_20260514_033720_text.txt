Multiple Access Links and Protocols
Two types of “links”:
1- point-to-point (direct connection):
A point-to-point link connects exactly two devices directly.There is no 
sharing of the connection with other devices.
point-to-point protocol (PPP) for dial-up access: Your computer
connects directly to the ISP over a phone line.
point-to-point link between Ethernet switch and host
(computer):Each computer has a dedicated cable to the switch — not 
shared with others.
Key idea:
Only two devices share the communication channel →no interference 
or competition.

Multiple Access protocols
Two or more simultaneous transmissions by nodes:
emerged problems: interference, QoS deterioration, security
collision if node receives two or more signals at the same time
Multiple access protocol concept
Distributed algorithm that determines how nodes share channel, i.e.,
determine when node can transmit
Communication about channel sharing must use channel
itself!
The devices must coordinate using the same channel they are
sharing.There is no separate (out-of-band) channel to manage 
coordination. So, the coordination itself must be efficient —
otherwise, it may cause more traffic and collisions.

1. Random Access Protocols
Examples: ALOHA, CSMA, CSMA/CD, CSMA/CA
Feature Description
Access Control Users compete for access to the medium randomly. No central control.
When to Transmit A station transmits whenever it has data, then checks for collisions.
Collision Handling Collisions can occur; detected or avoided depending on the protocol.
Efficiency Medium efficiency (depends on traffic load). Works well for low to moderate load.
Examples Explained « ALOHA - Simplest; send anytime and resend after random delay if collision.
+ CSMA - Sense channel before sending.
+ CSMA/CD - Detect collisions while transmitting (Ethernet).
+ CSMA/CA - Avoid collisions using acknowledgment and waiting (Wi-Fi).
Used in Ethernet (CSMA/CD), Wi-Fi (CSMA/CA).

2. Controlled Access Protocols
Examples: Reservation, Polling, Token Passing
Feature Description
Access Control Channel access is controlled in an orderly way — one station at a time.
Coordination Uses a central controller or distributed coordination (like token).
Collision Handling No collisions, since only one node transmits at a time.
Efficiency High efficiency under heavy load; less efficient under light load due to control overhead.
Examples Explained ¢ Reservation — Time slots are reserved before transmission.
* Polling — Central controller asks each node if it has data to send.
* Token Passing — A “token” circulates; only the node holding it can transmit.
Used in Token Ring, FDDI, Bluetooth piconets.

3. Channelization Protocols

Examples: FDMA, TDMA, CDMA
Feature Description
Access Control The channel is divided among users by frequency, time, or code.
Coordination Users have dedicated portions of bandwidth or codes — no contention.
Collision Handling No collisions (each user has its own “slice” of the channel).
Efficiency Very high for stable, continuous communication; less flexible for bursty traffic.
Examples Explained ¢ FDMA - Each user gets a unique frequency band.

* TDMA - Each user gets a time slot in a repeating frame.

- CDMA - All users share same frequency but use unique spreading codes.
Used in Cellular networks (GSM — TDMA; 3G — CDMA), satellite systems.

Multiple Access Protocols: Taxonomy
Three broad classes:
Channel Partitioning
Divide channel into smaller “pieces” (time slots “TDMA”,
frequency slots “FDMA”, uncorrelated codes “CDMA”)
Allocate piece to node for exclusive use “FDM”, “TDM”
Random Access
Channel not divided, allow collisions
“Recover” from collisions: use strategies like waiting a random time
before retransmitting (to reduce the chance of repeated collisions)
Controlled Access: “Taking turns”
Nodes take turns, but nodes with more to send can take longer turns
This type of network protocol, each node (device) gets a chance to use
the communication channel one at a time, avoiding collisions.
All nodes take turns sending data — like people taking turns to speak
in a conversation. However, nodes that have more data to send can be 
allowed to keep the channel longer before passing the turn to others. 
This ensures fairness (everyone gets a chance) while improving 
efficiency (busy nodes can send more without repeatedly waiting).

Random Access Protocols
Concept
When a node has a packet to send, it transmits immediately at the
full data rate (R) of the channel.
There is no prior coordination among nodes — every device decides
on its own when to send.
Because of this, if two or more nodes transmit at the same time,
their signals collide, and all packets involved become corrupted.
two or more transmitting nodes ➜“collision”
Random access MAC protocol :
The MAC (Medium Access Control) protocol defines how nodes
should:
1- Detect collisions – determine whether a transmission failed due 
to interference.
2- Recover from collisions – use strategies like waiting a random 
time before retransmitting (to reduce the chance of repeated 
collisions).

1.ALOHA (Areal Locations of Hazardous Atmospheres) – The simplest 
system; send whenever you have data. If collision occurs, wait a random 
time and resend.
2.Slotted ALOHA – Time is divided into slots; nodes can transmit only at 
the start of a slot, reducing collision probability.
3.CSMA (Carrier Sense Multiple Access) – A node listens (“senses”) 
the channel before transmitting to avoid collisions.
4.CSMA/CD (Collision Detection) – Used in Ethernet; the node listens 
while transmitting and stops immediately if a collision is detected.
5.CSMA/CA (Collision Avoidance) – Used in Wi-Fi; nodes use 
acknowledgments and waiting mechanisms to avoid collisions instead of 
detecting them.

1. Why CSMA/CA (Collision Avoidance) is used
e Unlike Ethernet (CSMA/CD) where devices can detect collisions, wireless stations cannot detect
collisions while transmitting because:
e The same antenna cannot transmit and listen simultaneously.
e Hidden terminals (stations out of range of each other) can cause undetectable collisions.
e So instead of collision detection (CD), Wi-Fi uses collision avoidance (CA).
2. Basic Process (Sender's Side)
(a) Sense the channel
e The station listens to the medium.
e If the channel is idle for a period called DIFS (Distributed Inter-Frame Space):
e — Transmit the entire frame immediately.
e There is no collision detection like in Ethernet.
(b) If the channel is busy
e The station waits until the channel becomes idle.
e Then it sets a random backoff timer.
e The timer counts down only while the channel is idle.
e If the channel becomes busy again, the countdown pauses.
(c) When the backoff timer expires
e The station transmits its frame.

3. Acknowledgment Mechanism
e When a receiver successfully receives a frame, it sends back an ACK (acknowledgment frame) after a
short interval, called SIFS (Short Inter-Frame Space).
e This ACK confirms successful reception.
4. What if No ACK is Received
e If the sender does not receive an ACK within a certain time:
e It assumes a collision occurred.
e The sender then increases the backoff interval (exponential backoff) and repeats the process from
step (b).
5. Timing Intervals
Interval Meaning Purpose
SIFS (Short Inter-Frame Space) Very short wait between data and Gives ACKs priority
ACK

Backoff Time Random waiting time Helps avoid simultaneous

6. Receiver’s Role
e The receiver waits for the entire frame.
e If received correctly (CRC check OK):
e It sends an ACK frame after SIFS.
e This ACK ensures the sender knows the transmission succeeded — crucial due to the hidden terminal
problem.

Overview
CSMA/CA is designed for collision avoidance (not detection like Ethernet CSMA/CD), because a wireless
station cannot listen and transmit at the same time.
The algorithm ensures that each device waits a random time before sending, reducing the chance that two
stations transmit simultaneously.
Step-by-Step Explanation
1. Start and Initialization
e The station has a frame to send.
e Initialize the attempt counter.
K = 0 (number of retransmission attempts).
2. Check if the Channel is Idle
e The station listens to the wireless medium.
e If the channel is busy, it waits until it becomes idle.
e If idle, it proceeds to the next step.
3. Wait for IFS Time
e The station waits for an Inter-Frame Space (IFS) period (usually DIFS in data frames).
e This ensures higher-priority transmissions (like ACKs or control frames) can go first.
4. Check if Still Idle
e After waiting IFS, if the channel is still idle, continue.
e If it becomes busy, go back to step 2 and monitor again.

Problem
In a shared wireless channel, if two nodes transmit large data 
frames at the same time, the frames collide, and a lot of time 
and bandwidth is wasted.
We want to avoid collisions, especially for long packets.
Idea: Reserve the Channel Before Sending Data
Instead of sending large data directly, the sender first sends a 
short control packet to request permission.
This is more efficient because:
•If a short packet collides →loss is small.
•Once permission is granted →no collision during the large 
transmission.

Token Passing (in Token Ring or Token Bus Networks)
In a token passing network, a special control frame called a token circulates around the network.
Only the device that possesses the token is allowed to send data.
How it works:
1. The token moves from one node to the next in a fixed, sequential order.
2. When a node has no data to send, it simply passes the token to the next node.
3. When a node wants to transmit data, it:
e Waits until it receives the token
e Grabs the token and sends its data frame
e After finishing, it releases the token back into the network so others can use it.
Key Terms
Term Meaning
Token A special small control packet that grants permission to transmit.
Token Message The act of sending/forwarding the token and the data frame while the node has
permission.

MAC Address Is a Flat Address — Portable

IP address — not portable (location-dependent).

Step-by-Step Process (ARP Operation)
1. A checks its ARP table
Host A looks in its ARP table for an existing entry:
e If it finds a mapping (B’s IP > B’s MAC), it can send the frame immediately.
e Ifo entry exists, ARP must be used to find the MAC address.
Each ARP table entry contains:
css
<IP address, MAC address, TTL>
e TTL = Time To Live (how long before the entry expires, usually ~20 minutes)
2. A broadcasts an ARP Request
A sends an ARP Request frame to the entire LAN:
e Destination MAC address: FF-FF-FF-FF-FF-FF (broadcast)
e¢ Content of ARP Request:
“Who has IP address of B? Tell A.”
Every device on the LAN receives the ARP Request because it is a broadcast.

1. Network Structure Overview
The diagram shows two local area networks (LANs):

e Left LAN: Contains host A and others connected to a switch or hub.

e Right LAN: Contains host B and others.

e Ris a router connecting both LANs.
2. Initial Conditions

e Host A wants to send an IP datagram to host B.

e Aknows B's IP address, but B is on a different network.

e Each LAN has its own network address (e.g., Net1 for A's LAN, Net2 for B’s LAN).
3. Determining Where to Send the Datagram
When A checks the destination IP (B's IP):

e It compares B's network prefix with its own.

e Since they are different, A realizes B is on another network.

e Therefore, A must send the datagram to its default gateway — the router R on its local network.
4. Resolving Router’s MAC Address (ARP)
To send the datagram to R, A needs R’s MAC address:

1. A checks its ARP table for the MAC address of the router interface on its LAN.

2. If not found, A sends an ARP request: “Who has [R’s IP]? Tell A.”

3. R responds with its MAC address.

4. A saves this info in its ARP table.
