1. What “Global Information” Means

In link-state routing, every router knows the entire network topology:
e All nodes (routers)
e All links between them
e Link costs (delay, bandwidth, congestion, etc.)

This is achieved through a process called link-state broadcast.

[Image 2]  685×679px  (PNG)
OCR:
Dijkstra’s Algorithm in Link-State Routing
Dijkstra’s algorithm computes least-cost paths from one router (“the source”) to all other routers.
How it works:
e Start at the source node.
e = Iteratively select the closest (minimum cost) unvisited node.
e Update the costs to its neighbors.
e Repeat until all nodes are included.
After running the algorithm, the router builds its forwarding table:
e For each destination > what is the next hop?
Iterative Nature of Dijkstra’s Algorithm
After k iterations, the algorithm knows:
e the least-cost path to k destinations
Eventually, when all nodes have been added, all shortest paths are known.
Table: Dijkstra’s Progress per Iteration
Iteration What Is Known
1 Shortest path to the closest neighbor
k Shortest path to k nodes
n Complete shortest-path tree

The Distance Vector (DV) algorithm is a routing algorithm used by routers to learn the best paths in a
network. It has three key properties:
1. Distributed
e Each router (node) does not know the entire network.
e Instead, it only knows:
e The costs to its directly connected neighbors
e The distance vectors received from those neighbors
e This means routing decisions are based on local information, not centralized control.
2. Iterative
e The algorithm runs repeatedly.
e Routers keep updating their routing tables whenever they receive new information.
e The process continues until no more changes occur in any router's table (convergence).
3. Asynchronous
e Routers do not update at the same time.
e Each router sends updates whenever there is a change, such as:
e Alink cost change
e Aneighbor sending an updated distance vector
e There is NO global clock forcing them to coordinate.

How It Works

From time to time, each router sends its distance vector to its neighbors.
A distance vector is simply a table listing:

The router's current estimate of the least-cost path to every destination.

When a router receives a distance vector from a neighbor, it updates its own table based on the Bellman-
Ford formula.

Bellman-Ford Update Rule

Let:

e  d,(y) = the cost of the least-cost path from router x to destination y.

e  ¢(x, v) = the direct link cost from x to its neighbor v.

e d,(y) = neighbor v's cost to get to y.
The update rule is:

d,(y) = min {e(a, v) + d,(y)}

Meaning:

For router x, the best way to get to destination y is to check every neighbor v and choose the path with the
smallest total cost:

(cost to reach neighbor v) + (cost from neighbor v to reach y)

This is applied repeatedly until the routing tables stabilize.

Example Use: RIP
The Routing Information Protocol (RIP) uses the Distance Vector approach with the Bellman-Ford
algorithm.

RIP Distance Vector Exchange — Explained
1. Routers exchange distance vectors every 30 seconds

e Each router periodically sends a RIP Response Message (also called a RIP advertisement).

e This happens every 30 seconds.

e The message is sent only to directly connected neighbors (because RIP is a distance vector protocol).
2. What is inside each RIP advertisement?
Each advertisement contains:

e Alist of up to 25 destination subnets (RIP limits each message to 25 entries).

e For each subnet, the router includes its distance (in hops) from itself to that subnet.
So a RIP message says something like:

“Here are all the networks | can reach, and here is how many hops away they are from me.”

The neighbor uses this information to update its own routing table using the Bellman-Ford rule.

[Image 2]  711×711px  (PNG)
OCR:
3. What happens if a router stops receiving advertisements?
RIP uses a timeout of 180 seconds.

e If router A does not hear any advertisement from a neighbor for 180 seconds:

e That neighbor is assumed dead.
e All routes that go through that neighbor become invalid.

This prevents routers from using stale routing information.
4. What happens after a neighbor is declared dead?
When routes become invalid:

1. The router updates its routing table.

2. It sends new advertisements to all its neighbors to announce the change.

3. Those neighbors check their own tables, update them if needed, and may also send updated

advertisements.

This event-triggered exchange helps the network quickly converge to new correct routes.
Why frequent updates + timers matter
RIP works with:

e Periodic updates (every 30 seconds)

¢ Timeout mechanism (180 seconds)

e Triggered updates (when a change is detected)
Together they allow routers to keep routing tables reasonably up-to-date despite RIP being a simple
protocol.

Item Explanation

Full Name Open Shortest Path First (open standard routing protocol)

Protocol Type Link-State routing protocol

Where Used Large enterprise networks and upper-tier ISPs (RIP mainly used in small
networks / lower-tier ISPs)

Routing Method Routers exchange Link-State Advertisements (LSAs) and flood them
through the AS

Topology Knowledge Each router builds a complete topology map of the network

Update Strategy LSAs sent on link state changes and periodically for refresh

LSA Content One entry per neighbor router with link cost and state

Packet Delivery OSPF runs directly over IP (not TCP or UDP)

Route Calculation Routers run Dijkstra's SPF algorithm to compute shortest paths

Result Creates optimized routing tables based on lowest path cost

Convergence Speed Fast, because updates are event-driven and calculations are local

Table 1 — What BGP Does

Function Explanation

Obtain reachability info Each Autonomous System (AS) learns which IP subnets are reachable through its
neighboring ASs.

Propagate routes internally Border routers share the learned reachability information with all other routers
inside the AS (using IGP such as OSPF).

Select “good” routes BGP chooses the best paths using routing policies and attributes (not shortest
distance).

Advertise subnet existence A subnet announces itself to the Internet: “I am here — this is how to reach me.”

Global connectivity Makes it possible for all networks worldwide to learn routes to each other.

Why Source Duplication is Inefficient

Issue Explanation

Redundant traffic Many identical packets flood the network even when fewer copies are sufficient.

Bandwidth waste Duplicate transmissions unnecessarily consume link capacity.

Processing overhead Routers must process repeated copies of the same packet.

Possible network congestion Excessive duplicates increase queuing delay and packet loss.

No loop control Without intelligent forwarding, packets can loop back and return to routers that
already forwarded them.

Controlled Flooding & Spanning Tree Broadcast

Technique Explanation

Controlled Flooding Routers forward a broadcast packet only if they have not forwarded that same
packet before (tracked using packet IDs or sequence numbers).

Result Prevents infinite loops and significantly reduces duplication compared to simple
flooding.

Spanning Tree Broadcasting A loop-free tree structure covering all nodes is created. Each broadcast packet is
forwarded only along tree links.

Benefit Guarantees no redundant packets — each node receives exactly one copy.

Join Process Each node sends a unicast “join” message to a center (root) node to build the
spanning tree.

Overall Efficiency Maximum efficiency with guaranteed full coverage and no duplicates.

Comparison Summary

Method Efficiency Duplicates? Safety

Flooding Low Yes — many Risk of congestion

Controlled Flooding Medium Few Safe from loops

Spanning Tree High No duplicates Fully safe
