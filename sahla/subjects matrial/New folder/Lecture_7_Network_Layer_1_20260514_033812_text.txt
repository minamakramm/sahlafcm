Explanation: Introduction to the Network Layer
The network layer (Layer 3) is responsible for moving data from the sending host to the receiving host
across multiple networks. It operates in both end systems (hosts) and intermediate systems (routers).
Key Functions:
1. Encapsulation at the Sender
e The transport layer creates segments (TCP/UDP).
e The network layer takes each segment and encapsulates it into a datagram (IP packet).
e These datagrams are then delivered to the data link layer to be transmitted across the physical network.
2. Forwarding and Routing Across Routers
e Every router runs network-layer protocols (like IP).
e When a datagram arrives, the router examines the IP header:
e It reads the destination IP address.
e Based on its routing table, it decides where to forward the datagram next.
¢ The datagram may pass through many routers before reaching the destination.
3. Delivery to the Transport Layer at the Receiver
e The final-hop router sends the datagram to the receiving host.
e The network layer at the receiving host:
e Removes the IP header.
e Passes the encapsulated segment upward to the transport layer.

1. Packet arrives at the router

OCR:
A service model defines what the network guarantees when it carries data (datagrams) from a sender to a

receiver.

1. Services for Individual Datagrams

These guarantees apply to each datagram separately.

Service Meaning

Guaranteed delivery The network promises the datagram will reach the destination (no loss).

Guaranteed delivery with a delay The datagram will arrive within a maximum delay—e.g., < 40 ms. Useful for real-

bound time traffic (voice, video).

2. Services for a Flow of Datagram

These guarantees apply to a stream (flow) of datagrams belonging to the same communication session:

Service Meaning

In-order delivery Datagrams arrive in the same sequence they were sent.

Guaranteed minimum bandwidth The network reserves a certain data rate (e.g., 2 Mbps) for the entire
flow — avoids congestion.

Summary (1 sentence)

The service model specifies what delivery guarantees (like reliability, delay, ordering, and bandwidth) the

network provides for datagrams or flows traveling between sender and receiver.

Summary
In a Virtual Circuit (VC) network, each router uses a forwarding table to decide how to handle packets.
e A packet arrives on an incoming interface with an incoming VC number.
e The router looks up this (interface, VC) pair in its forwarding table.
e The table specifies an outgoing interface and a new VC number.
e The router replaces the old VC number with the new one and forwards the packet on the specified
interface.
Because of this, routers maintain connection state and each hop can use different VC numbers.

The figure shows how a router forwards packets in the IPv4 datagram (Internet) model based on the
destination IP address.
1. IPv4 Addresses Are 32 Bits
An |Pv4 address has 32 bits, meaning there are over 4 billion possible addresses
— This is why the note says “4 billion possible entries.”
A router must decide where to forward a packet based on these bits.
2. Forwarding Table Structure
The table contains destination address ranges, each mapped to a specific outgoing link interface.
Example entries:
e If the destination address falls in the blue range, send it to interface 0
e If it falls in the green range, send it to interface 1
e If it falls in the red range, send it to interface 2
e Otherwise, send it to interface 3 (a default route)
3. How the Router Uses the Table
When a packet arrives:
1. The router checks the packet's destination IP address.
2. It compares the address to the ranges in the table.
3. It forwards the packet out the interface associated with the matching range.
This is called longest-prefix matching in IP routing, though not shown explicitly here.
